架构师

Author: 杨佳男 Date: 2019-08-27  07:31:05 Category: 行业知识

架构师类别

业务架构师（Business Architect）

业务架构师根据运营需求解释和背景化战略，开发特定的人工制品，如业务能力图和价值流，以帮助弥合战略与执行之间的差距，并帮助简化和理顺IT启用流程。除了开发特定的交付物、观点和观点外，架构师还综合和协同他人的工作，从战略制定、业务分析、流程管理、运营和系统分析等方向制定蓝图。

主题领域架构师（Domian Architect）

技术架构师（Technology Architect）

项目架构师（J2EE.NET架构师）

系统架构师（System Architect）

系统架构师是系统或产品线的设计负责人，是一个负责理解和管理并最终确认和评估非功能性需求，给出开发规范，搭建系统实现的核心架构，对整个软件架构、关键构件、接口进行总体设计并澄清关键技术细节的高级技术人员。是信息系统开发和演进的全方位技术和管理人才。

具备技术素质
  • 丰富的一线中大型开发项目的总体规划、方案设计及技术队伍管理经验。
  • 软件行业工作经验，熟悉业务领域的技术应用和发展。
  • 项目管理理论基础，并在应用系统开发平台和项目管理上有实践经验。
  • 对技术标准有深刻的认识，对软件工程标准规范有良好的把握。
  • 具备C/S或B/S体系结构或特定领域软件产品开发及架构和设计的经验。
  • 面向对象分析，设计、开发能力，精通UML和XML，熟练使用Rational Software Architect、PowerDesigner等CASE工具进行设计开发。
  • 精通架构和设计模式。
  • 精通数据库。
  • 对计算机系统，网络安全，应用系统架构有全面的认识。
  • 良好的团队意识和协作精神，有较强的内外沟通能力。
具备管理素质
  • 远见、诚信、果断地领导素质。
  • 使所有团队人员的想法保持一致，为一个项目制订一个清晰的、强制性的、有远见的目标作为整个团队的动力，从而达到整体目标所作的权衡提供基础。
  • 必须提供特定的方法和模型作为理想的技术解决方案。
  • 及时解决技术问题的紧迫感和自信心。

具备多层次+多方面的知识维度，很强的表述能力，技术、管理、心理和艺术等多方面的知识和能力，多个角色的特点。
