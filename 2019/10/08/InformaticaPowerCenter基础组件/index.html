InformaticaPowerCenter基础组件

Author: 杨佳男 Date: 2019-10-08  19:14:48 Category: Informatica

数据库源 Source

数据库目标 Target

行级表达式的计算Expression

对某个字段进行四则运算，对某些字符串进行计算等，可定义输入输出，变量

过滤组件 Filter

满足条件的数据通过这个组件，不满足条件的数据会被该组件过滤掉。

数据类型转换 Source Qualifier

数据源拖入之后，会出现两个组件，将数据库类型长度转换为相同数据类型

可以在SQ中对表进行关联，限制，分组

排序 Sorter

可限制是否大小写敏感，实现Distinct，是否将NULL视作最小值 一般使用在Joiner之前

关联 Joiner

定义关联条件 最好将Master方为小表 ，连接类型NJ(inner join)，MOJ(left join)，DOJ(rigth join)，FOJ(full join) 在Source Qualifier中也可进行关联，但是关联运行的环境不同

自关联Self-Join

同一张表进行自我关联才能得到期望的结果

在从Source Qualifier到Joiner的两个分支上，至少一个分支必须在在Source Qualifier和Joiner之间增加一个其他组件

Joiner Key必须是经过排序的

Joiner需要使用Sorter

查询关联 Lookup

可用于非连接，需添加关联条件，若担心排序增加数据库压力，则可在生成语句时将排序备注掉

可使用Lookup进行非插即更逻辑处理，

存储过程 Stroed Procedure

可以调用数据库中的存储过程，也可分为连接和非连接，使用时session配置需要给Stored Procedure提供数据库连接，这个连接指向的是存放存储过程的数据库。

合并 Union

将两张表或者多张表联合到一起。提供Distinct功能

事务控制 Transaction Control

控制事务的提交和回滚，定义或重新定义事务的边界，清除了任何来自上游的事务。

提供了5个支持事务控制的常量：

TC_CONTINUE_TRANSACTION 默认值 不进行任何事务变更。

TO_COMMIT_BEFORE 提交当前事务，开始一个新的事物，将当前事务的数据写入目标表。当前行进入新事务中。

TO_COMMIT_AFTER 提交当前事务，开始一个新事务，将当前事务的数据写入目标表。当前行在本事务中。

TO_ROLLBACK_BEFORE 回滚当前事务，开始新事物。当前行归入新事物中。

TO_ROLLBACK_AFTER 回滚当前事务，开始新事物。当前行归入回滚事务中。

可能导致Transaction Contral失效的组件：Aggregator、Joiner、Rank、Sorter、Custom transformation

序列 Sequence

自增长的序列，通常用于要生成数据纪录的自增长或序号。多个session操作同一数据集，使用Cached Vlues来保证sequence不会产生冲突。

数据聚合 Aggregator
完成SUM\AVG\MAX等聚合运算。可以根据指定条件进行汇总，如SUM(SAL,SAL>900)，可以实现行列转换。

排序 Rank

用于实现Top或Bottom的数据，选择前几或最后几个，在配置中勾选R列来选择排序的列，还可增加分组字段

更新策略 Update strategy

进行update和delete操作，有两种方法，使用session的属性Treat source rows as 默认是insert，也可以使用该组件指定data drivrn，设置表达式，不同数据不同处理。可以使用Lookup组合时限update else insert操作。

执行SQL语句 SQL Transformation

包含两种模式Script Mode和Query Mode，Query Mode分为Static Query Mode和Dynmic Query Mode

Java Transformation

可以使用Java代码处理一些特殊的业务场景，比如判断字段是否含有中文、汉字转拼音、加密/解密、行专列等内置组件不能完成的。

列转行 Normalizer

数据分发 Router

扩展组件 Custom Transformation

HTTP Transformation

可以通过HTTP协议连接到HTTP SERVICE使用其服务和应用的能力。功能包括更新和获取HTTP服务器上的数据。包含三种授权方式：Basic，Digest，NTLM**

Transformation中的概念

  • connect与unconnect： 与其它组件直接通过连接线进行连接还是不与其他组件连接。

  • active与passive ： 输入输出数据相同条数则为active，否则为passive
